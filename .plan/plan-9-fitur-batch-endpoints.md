# Role & Context

You are an expert Laravel 13 developer. Our database now uses ULIDs. The frontend is an offline-first mobile app. We need a batch synchronization endpoint.

# Task: Create a `/api/sync/transactions` endpoint to handle bulk inserts/updates from the mobile client.

# Requirements:

1. Create a `SyncController` with a `transactions` method.
2. Route: `POST /api/sync/transactions` protected by `auth:sanctum`.
3. The endpoint will receive a JSON payload with this structure:
    ```json
    {
        "transactions": [
            {
                "id": "01ARZ3NDEKTSV4RRFFQ69G5FAV", // ULID generated by mobile app
                "wallet_id": "...",
                "transaction_category_id": "...",
                "amount": 50000,
                "name": "Lunch",
                "note": null,
                "type": "outcome", // to help determine balance logic
                "created_at": "2026-04-29T12:00:00Z", // Mobile app timestamp
                "deleted_at": null // If not null, it means mobile app soft-deleted it
            }
        ]
    }
    ```
4. Controller Logic (CRITICAL):
    - Iterate through the `transactions` array inside a `DB::transaction()`.
    - Use `Transaction::updateOrCreate()` or `upsert()` keyed by the ULID (`id`).
    - Idempotency: If the transaction already exists in the DB, do NOT mutate the wallet balance again unless the amount changed (handle balance delta). If it's a new transaction, mutate the wallet balance accordingly (income adds, outcome subtracts).
    - If `deleted_at` is provided from the client, soft delete the record on the server and reverse the wallet balance.
5. Create a `SyncTransactionRequest` to rigorously validate the batch payload. Check cross-user foreign keys (`wallet_id`, `category_id`) to ensure the user owns them.
6. Write rigorous Pest tests for `SyncController` verifying:
    - Successful batch insertion.
    - Idempotency (sending the same batch twice doesn't double-charge the wallet).
    - Validation failures for missing/invalid data.
7. Add dedoc/scramble PHPDoc blocks to document the expected batch payload.
